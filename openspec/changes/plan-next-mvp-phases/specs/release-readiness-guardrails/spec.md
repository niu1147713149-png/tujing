## ADDED Requirements

### Requirement: 生成前必须暴露基础可用性状态
系统 MUST 在用户提交生成任务前尽早发现后端不可用、模型配置缺失或无效模型选择，避免把错误延迟到结果页。

#### Scenario: 后端健康检查失败
- **WHEN** 生成页检测到后端健康检查失败
- **THEN** 系统 MUST 阻止提交生成，并提示用户先确认前后端服务状态

#### Scenario: 模型配置无效
- **WHEN** 用户提交了不存在或不可用的模型配置
- **THEN** 系统 MUST 返回明确的简体中文错误信息，说明问题来自模型配置而不是泛化为未知错误

### Requirement: 发布前必须覆盖核心任务链路烟测
系统 SHALL 在发布或重要迭代前，对首页、生成页、结果页和任务接口执行可重复的核心链路验证。

#### Scenario: 本地烟测通过核心链路
- **WHEN** 运行项目约定的烟测流程
- **THEN** 流程 MUST 覆盖页面可打开、`/api/tasks` 可达、生成按钮可创建任务，以及结果页进入轮询状态

#### Scenario: 页面文案验收
- **WHEN** 完成涉及用户可见内容的改动
- **THEN** 验收 MUST 检查首页、生成页、结果页和相关 API 错误文案中不存在乱码、问号占位或非预期英文主文案

### Requirement: 规划文档必须持续匹配当前架构
系统 SHALL 确保后续路线图、规范和说明文档基于当前 `frontend/` + `backend/` 架构，而不是引用已归档的一体化目录。

#### Scenario: 更新后续开发文档
- **WHEN** 团队编写新的路线图、规范或任务计划
- **THEN** 文档 MUST 以 `frontend/`、`backend/`、`config/`、`docs/` 为当前有效边界，不得再把旧版 Next.js 路径当作现行实现
