## ADDED Requirements

### Requirement: 订单成为生成工作的主入口
系统 MUST 以订单作为用户进入生成工作台、继续创作和回看历史的主入口，而不是让任务列表成为唯一入口。

#### Scenario: 从首页或历史页新建订单
- **WHEN** 用户在首页或历史页触发“新建订单”
- **THEN** 系统 MUST 创建订单记录，并把用户带到该订单对应的生成工作台

#### Scenario: 从历史记录继续回到订单上下文
- **WHEN** 用户在历史页打开某个已有订单
- **THEN** 系统 MUST 允许用户进入最近任务结果页，或在没有历史任务时直接回到该订单工作台

### Requirement: 订单历史必须支持继续处理而不是只读查看
系统 SHALL 在历史记录中展示足够上下文，让用户能判断是否继续生成、查看最近结果或删除旧订单。

#### Scenario: 历史卡片展示关键上下文
- **WHEN** 历史页加载订单列表
- **THEN** 每张订单卡片 MUST 展示订单备注、模板信息、任务数量、成功数、失败数，以及最近任务的模型或提示词摘要

#### Scenario: 删除订单前进行明确确认
- **WHEN** 用户删除订单
- **THEN** 系统 MUST 明确说明会删除该订单下的历史任务和已生成图片，且删除后无法恢复
