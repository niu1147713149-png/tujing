# 图鲸 MVP 商业化规划（1-3 个月）

## 1. 当前阶段定义

图鲸当前处于 **MVP 任务型生成流** 阶段，核心价值链路已验证：

- 用户选择模板 → 输入 prompt → 创建任务 → 轮询结果 → 下载图片

但距离稳定商业化还需补齐：

- 核心流程的稳定性与可靠性
- 用户价值感知与转化路径
- 商业化能力（付费、限制、升级提示）
- 基础数据埋点与监控
- 用户留存与反馈机制

本阶段规划重点不是扩展新功能，而是围绕 **"让用户愿意持续使用，并愿意付费"** 展开。

---

## 2. 阶段目标

### 2.1 业务目标
- 完成 MVP 到可商业化版本的过渡
- 建立清晰的付费路径与定价策略
- 验证首批真实付费用户
- 提升核心场景下的使用频率与转化率

### 2.2 产品目标
- 打磨核心主流程，降低使用门槛
- 优化首屏、生成页、结果页的转化效率
- 增强用户对产品价值的感知
- 完善异常状态与错误提示

### 2.3 技术目标
- 保证核心链路稳定性与可靠性
- 建立基础埋点与数据看板
- 控制技术债增长
- 为后续迭代留出扩展空间

---

## 3. 规划原则

1. **围绕商业化闭环展开**
   - 所有功能优先级都以"是否影响转化/留存/付费"为判断标准

2. **优先修复阻碍使用的问题**
   - 任何影响用户完成核心任务的问题都优先于新功能

3. **避免过度扩展**
   - 1-3 个月周期内只做能直接推动商业化的内容

4. **数据驱动迭代**
   - 通过埋点、反馈、转化数据来决定下一步优化方向

## 6. 商业化策略

### 6.1 定价策略建议

建议先用低复杂度方案验证：

**方案 A：免费试用 + 付费升级**
- 免费：5 次生成 / 月
- 付费：¥9.9 / 月（50 次）或 ¥29.9 / 月（无限）

**方案 B：次数包 + 月度订阅**
- 按次购买：¥0.5 / 次（最少 10 次）
- 月度订阅：¥9.9 / 月（50 次）

**方案 C：增值功能解锁**
- 基础功能免费（标准模板、标准分辨率）
- 高级功能付费（高级模板、高分辨率、优先队列）

### 6.2 收费判断标准

以下任一条件满足即可考虑收费：
- 用户明确愿意为结果付费
- 产品替代了明确的人工成本或第三方工具成本
- 核心功能具备持续使用价值
- 付费后可显著提升效率或效果

### 6.3 商业化目标

- 验证首批真实付费用户（目标：第 6 周 10+ 付费用户）
- 明确用户对价格的接受区间
- 找到最有效的收费方式
- 建立可持续的商业模式

## 7. 数据与指标

### 7.1 漏斗指标
- 访问数 / 日
- 注册率（注册用户 / 访问）
- 激活率（完成首次生成 / 注册）
- 核心功能使用率（生成任务数 / 激活用户）
- 付费转化率（付费用户 / 激活用户）

### 7.2 留存指标
- 次日留存（Day 1 / Day 0）
- 7 日留存（Day 7 / Day 0）
- 30 日留存（Day 30 / Day 0）
- 复访率（周活 / 月活）

### 7.3 商业指标
- 付费用户数
- ARPU（平均每用户收入）
- ARPPU（平均每付费用户收入）
- 试用转付费率
- 退款率 / 流失率

### 7.4 体验指标
- 关键流程完成时长（从首页到结果页）
- 错误率（任务失败率）
- 客诉/反馈数量
- 支付成功率

## 8. 风险与应对

### 风险 1：功能扩张过快，偏离商业化重点
**应对：**
- 所有需求先过商业价值判断
- 只保留能推动转化的功能
- 定期评估功能与商业目标的关联度

### 风险 2：付费链路不顺畅，转化率低
**应对：**
- 优先保证购买路径简单清晰
- 减少付费前的阻力（如：不要求过多信息）
- 快速迭代价格与套餐设计

### 风险 3：没有真实用户反馈，决策盲目
**应对：**
- 主动收集用户使用反馈
- 定期进行用户访谈
- 快速迭代，不等"大版本"

### 风险 4：数据不可观测，无法判断方向
**应对：**
- 提前埋点，不要等到后期补
- 建立最小可用看板
- 定期分析数据，指导决策

### 风险 5：Gemini 接入不稳定，影响用户体验
**应对：**
- 完善错误处理与重试机制
- 建立降级方案（如：显示友好错误提示）
- 监控 API 调用成功率

## 10. 进入下一阶段的判断标准

当以下条件中有多个被满足时，可进入下一阶段规划：

- 已有稳定的核心使用路径（任务成功率 > 95%）
- 出现真实付费用户（> 10 个）
- 转化链路稳定可复用（转化率稳定在某个区间）
- 用户反馈开始呈现明确模式（高频问题清晰）
- 产品有清晰的增长入口（某个获客方式有效）

## 12. 当前执行原则

- **先稳定，再扩展** — 核心流程稳定是一切的基础
- **先一致，再美化** — 按 PROJECT_STANDARDS.md 保持设计语言一致
- **先任务流，再高级功能** — 不偏离任务型生成流的核心定位
- **先中文正确，再视觉精修** — 文案与编码问题必须优先修
- **发现可见乱码，必须继续修，不得忽略** — 这是用户体验的底线
